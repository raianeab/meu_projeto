API de Gerenciamento de Eventos

Esta aplicação expõe endpoints REST para gerenciamento de usuários, eventos, categorias, inscrições e certificados. Todos aceitam e devolvem JSON.

1. Usuários

  • GET /users – lista todos os usuários.
  • POST /users – cria usuário.
    Exemplo de corpo: {"nome_completo":"Ana Silva","tipo_usuario":"participante"}
  • GET /users/:id – detalhe do usuário.
  • PUT /users/:id – atualiza usuário.
    Exemplo de corpo: {"nome_completo":"Ana Silva","tipo_usuario":"organizador"}
  • DELETE /users/:id – remove usuário.

2. Categorias

  • GET /categories – lista todas as categorias.
  • POST /categories – cria categoria.
    Exemplo de corpo: {"nome_categoria":"Workshop","descricao":"Eventos práticos"}

3. Eventos

  • GET /events – lista todos os eventos.
  • POST /events – cria evento.
    Exemplo de corpo: { "titulo": "Workshop de Programação", "descricao": "Aprenda programação do zero", "local": "Auditório Principal", "data_inicio": "2024-06-01T09:00:00Z", "data_fim": "2024-06-01T18:00:00Z", "tipo_evento": "workshop", "vagas_disponiveis": 30, "publico_alvo": "Iniciantes em programação", "id_organizador": "uuid-do-organizador" }

4. Inscrições

  • GET /inscriptions – lista todas as inscrições.
  • POST /inscriptions – cria inscrição.
    Exemplo de corpo: { "id_evento": "uuid-do-evento", "id_usuario": "uuid-do-usuario" }

5. Certificados

  • GET /certificates – lista todos os certificados.
  • POST /certificates – cria certificado.
    Exemplo de corpo: { "id_inscricao": "uuid-da-inscricao", "caminho_arquivo": "/certificados/certificado.pdf", "data_emissao": "2024-06-01T18:00:00Z" }

6. Categorias de Eventos

  • GET /event-categories – lista todas as categorias de eventos.
  • POST /event-categories – associa categoria a evento.
    Exemplo de corpo: { "id_evento": "uuid-do-evento", "id_categoria": "id-da-categoria" }

7. Feedbacks de Eventos

  • GET /event-feedbacks – lista todos os feedbacks.
  • POST /event-feedbacks – cria feedback.
    Exemplo de corpo: { "id_evento": "uuid-do-evento", "id_usuario": "uuid-do-usuario", "avaliacao": 5, "comentario": "Excelente evento!" }

Exemplos de uso via curl

# Criar um usuário
curl -X POST http://localhost:3000/users \
     -H "Content-Type: application/json" \
     -d '{"nome_completo":"Ana Silva","tipo_usuario":"participante"}'

# Criar uma categoria
curl -X POST http://localhost:3000/categories \
     -H "Content-Type: application/json" \
     -d '{"nome_categoria":"Workshop","descricao":"Eventos práticos"}'

# Criar um evento
curl -X POST http://localhost:3000/events \
     -H "Content-Type: application/json" \
     -d '{
       "titulo": "Workshop de Programação",
       "descricao": "Aprenda programação do zero",
       "local": "Auditório Principal",
       "data_inicio": "2024-06-01T09:00:00Z",
       "data_fim": "2024-06-01T18:00:00Z",
       "tipo_evento": "workshop",
       "vagas_disponiveis": 30,
       "publico_alvo": "Iniciantes em programação",
       "id_organizador": "uuid-do-organizador"
     }'
